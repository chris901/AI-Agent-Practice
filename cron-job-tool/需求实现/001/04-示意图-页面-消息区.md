# 04 — 示意图：页面 · 消息区模块

## 功能对齐（需求）

- 用户 / 助手分区、时间顺序；流式输出；结束后与权威历史一致（《技术方案》§3.5）。

---

## ASCII — 静态时间线

```
消息区 (可滚动)
|
|   +----------------------------------------+
|   | 用户                                    |
|   | 请总结上周会议结论。                     |
|   +----------------------------------------+
|                      +----------------------------------------+
|                      | 助手                                    |
|                      | 结论主要有三点：一、... 二、... 三、...  |
|                      +----------------------------------------+
|   +----------------------------------------+
|   | 用户                                    |
|   | 第三点展开。                             |
|   +----------------------------------------+
|                      +----------------------------------------+
|                      | 助手                                    |
|                      | 第三点指的是预算审批需补充附件...        |
|                      +----------------------------------------+
v
(底部接输入区)
```

---

## ASCII — 流式输出中（助手气泡内）

```
                      +----------------------------------------+
                      | 助手                                    |
                      | 第三点指的是预算审批需补充附件，目前█     |
                      +----------------------------------------+
                                                    ^ 光标/闪烁
```

---

## ASCII — 流式结束 → 与列表对齐（概念）

```
  SSE 推送增量文本 → 前端 buffer 展示
           |
           v
  收到结束 / 连接关闭
           |
           v
  以服务端为准：重新 GET 消息列表 或 合并完整一条助手消息
           |
           v
  时间线展示与数据库一致
```

---

## ASCII — 长消息折叠（可选，纯前端）

```
+----------------------------------------+
| 助手                                    |
| 很长很长很长...(前 5 行)                |
| [ 展开全文 v ]                          |
+----------------------------------------+
```
